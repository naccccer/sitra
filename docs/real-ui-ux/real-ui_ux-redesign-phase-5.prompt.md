# Real UI/UX Redesign Phase 5 Prompt

## Objective
Close the redesign program by removing residual legacy patterns, finalizing governance, and documenting how the unified system stays healthy over time.

## Scope
In scope:
- Remove leftover pre-redesign UI patterns and dead variants
- Close remaining consistency gaps across modules
- Finalize contribution, review, and maintenance documentation
- Define the lightweight governance loop that prevents future fragmentation
- Publish the final KPI summary, remaining debt, and rewrite triggers

Out of scope:
- New net-new redesign directions that belong to a future program
- Broad feature work unrelated to consistency closure or governance
- Historical cleanup beyond what affects operational clarity

## Locked decisions
- The app should exit the program with one unified visual and interaction language.
- Governance should be lightweight but enforceable.
- Historical redesign material should not be carried forward unless a future task explicitly requires it.
- The final system should be maintainable through normal product work, not another large rewrite immediately after close.

## Required repo and doc updates
- Finalize the roadmap close-out sections.
- Finalize contribution and governance docs that future UI changes must follow.
- Remove outdated redesign prompts, notes, and deprecated UI pattern references that are no longer needed.

## Implementation checklist
- Remove dead shared variants and leftover bespoke patterns.
- Close any remaining cross-module consistency gaps that are still visible in active workflows.
- Finalize governance language for shared primitives, tokens, states, and review expectations.
- Publish final KPI summary and unresolved debt with clear ownership.
- Define rewrite triggers or maintenance thresholds so future drift is caught early.

## Verification checklist
- Active modules no longer present obvious pre-redesign visual or interaction leftovers.
- Governance and contribution docs are concise, current, and enforceable.
- Final KPI summary is present and traceable to earlier baseline reporting.
- Remaining debt is explicit rather than hidden in the runtime.

## Acceptance criteria
- The redesign closes with one coherent product language across active operational surfaces.
- Future contributors can understand how to maintain the system without reopening major ambiguity.
- The repo no longer carries stale active redesign instructions from previous cycles.

## Roadmap update requirements at phase close
Update `real-ui_ux-redesign-roadmap.md` with:
- What changed in the final cleanup and governance pass
- What remains as accepted debt
- Final KPI summary
- Residual risks or rewrite triggers
- The final operating model for post-program maintenance
